1. What is Missing Data?

Missing data occurs when no value is stored for a variable in an observation. In pandas it is represented as NaN (Not a Number), None, pd.NA, or pd.NaT (for datetimes).

NaN
Float-based sentinel from NumPy. Most common representation for numeric columns.
import numpy as np; np.nan
None
Python object, used for object dtype columns. pandas converts it to NaN in numeric context.
df.loc[0, 'col'] = None
pd.NA
Introduced in pandas 1.0 for nullable integer, boolean, and string dtypes.
pd.array([1, pd.NA, 3], dtype="Int64")
pd.NaT
Not a Time — sentinel for missing values in datetime, timedelta columns.
pd.Timestamp(pd.NaT)

Setup — the dataset we use throughout this guide

2. Detecting Missing Data

Always identify the extent and pattern of missingness before deciding how to handle it.

isnull() / isna() — per-cell boolean mask

Equivalent methods 
# Boolean mask — True where value is missing
print(df.isnull())

# Count missing values per column
print(df.isnull().sum())

# Percentage missing per column
print((df.isnull().sum() / len(df)) * 100)

# Total missing across entire DataFrame
print(df.isnull().sum().sum())

3. Dropping Missing Data

Use when missing rows or columns carry too little information to be salvaged, or when the dataset is large enough that removal is safe.

dropna() — rows and columns

# Drop rows with ANY missing value
df_no_na = df.dropna()
print(f"Original: {len(df)} rows  →  After drop: {len(df_no_na)} rows")

# Drop rows only if ALL values are missing
df_drop_all = df.dropna(how="all")

# Drop rows where specific critical columns are missing
df_critical = df.dropna(subset=["name", "age"])
print(df_critical[["patient_id", "name", "age"]])

# Drop rows where fewer than 5 valid values exist
df_thresh = df.dropna(thresh=5)
print(f"After thresh=5: {len(df_thresh)} rows")

# Drop columns with any missing value
df_col_drop = df.dropna(axis=1)
print(df_col_drop.columns.tolist())

⚠️ Warning: Dropping rows introduces survivorship bias. If patients with missing age are systematically older (less likely to self-report), dropping them skews your analysis. Always check why data is missing.

4. Filling & Imputation

Replace missing values with a meaningful substitute instead of discarding records.

fillna() — constant or computed fill

💊 Scenario: Fill missing patient age with median age, missing blood type with "Unknown". 
# Fill a single column with its median
median_age = df["age"].median()       # → 36.0
df["age"] = df["age"].fillna(median_age)

# Fill categorical column with placeholder
df["blood_type"] = df["blood_type"].fillna("Unknown")

# Fill multiple columns with different values in one call
df = df.fillna({
    "cholesterol": df["cholesterol"].mean(),
    "name":         "Patient Unknown",
})

print(df["age"].isnull().sum())       # → 0
print(df["blood_type"].value_counts())

Forward-fill (ffill) & Back-fill (bfill)

📈 Scenario: Daily IoT temperature sensor readings — a sensor briefly disconnects. Propagate the last known reading forward. 
sensor = pd.DataFrame({
    "timestamp": pd.date_range("2024-06-01", periods=8, freq="h"),
    "temp_c": [22.1, np.nan, np.nan, 22.8, 23.0, np.nan, 23.5, 23.7],
})

# Forward fill: each NaN takes the previous valid value
sensor["temp_ffill"] = sensor["temp_c"].ffill()

# Back fill: each NaN takes the next valid value
sensor["temp_bfill"] = sensor["temp_c"].bfill()

# Limit propagation to at most 1 consecutive NaN
sensor["temp_ffill_lim"] = sensor["temp_c"].ffill(limit=1)

print(sensor)
Output

1 2024-06-01 01:00:00     NaN        22.1        22.8             NaN  ← limit=1 stops here
2 2024-06-01 02:00:00     NaN        22.1        22.8             NaN

💡 Tip: Use limit= to prevent blindly propagating stale values across many rows. If a sensor has been down for 6 hours, a forward-filled value 6 hours old is probably misleading.

Group-based Imputation (conditional mean/mode)

🛒 Scenario: E-commerce dataset — product price is missing for some rows. Fill with the median price within the same product category. 
ecom = pd.DataFrame({
    "product":  ["Laptop", "Laptop", "Phone", "Phone", "Tablet", "Tablet"],
    "category": ["Electronics"] * 4 + ["Electronics", "Electronics"],
    "price":    [999, np.nan, 699, np.nan, 499, 449],
    "rating":   [4.5, 4.3, np.nan, 4.1, 4.7, np.nan],
})

# Fill price with median price of the same product
ecom["price"] = ecom.groupby("product")["price"].transform(
    lambda x: x.fillna(x.median())
)

# Fill rating with mean rating within category
ecom["rating"] = ecom.groupby("category")["rating"].transform(
    lambda x: x.fillna(x.mean().round(2))
)

print(ecom)

5. Interpolation

Estimate missing numeric values based on surrounding data points. Best for sequential or time-series data.

interpolate() — multiple methods

📊 Scenario: Monthly sales figures with some months missing due to a reporting system outage. 
sales = pd.DataFrame({
    "month": pd.date_range("2024-01", periods=10, freq="ME"),
    "revenue": [50000, np.nan, np.nan, 62000, 67000,
               np.nan, 72000, 75000, np.nan, 80000],
}).set_index("month")

# Linear interpolation (default) — equal spacing between neighbours
sales["linear"]  = sales["revenue"].interpolate(method="linear")

# Polynomial order-2 — fits a curve through surrounding points
sales["poly2"]   = sales["revenue"].interpolate(method="polynomial", order=2)

# Spline (cubic) — smooth curve, good for trends
sales["spline"]  = sales["revenue"].interpolate(method="spline", order=3)

# Time-aware interpolation (requires DatetimeIndex)
sales["time"]    = sales["revenue"].interpolate(method="time")

print(sales.round(0))

6. Advanced Techniques

Statistical and ML-based approaches for complex missingness patterns.

KNN Imputation (scikit-learn)

🏡 Scenario: Housing dataset — square footage and number of bedrooms are correlated. Use similar houses (neighbours) to impute missing values. 
from sklearn.impute import KNNImputer

housing = pd.DataFrame({
    "sqft":     [1500, 2000, np.nan, 1800, 2200, np.nan],
    "bedrooms": [3, 4, 3, np.nan, 4, 5],
    "price_k":  [300, 450, 320, 380, np.nan, 520],
})

imputer = KNNImputer(n_neighbors=2)
imputed_array = imputer.fit_transform(housing)
housing_imputed = pd.DataFrame(imputed_array, columns=housing.columns)
print(housing_imputed.round(1))
Output

0  1500.0       3.0    300.0
1  2000.0       4.0    450.0
2  1650.0       3.0    320.0   ← sqft imputed from 2 nearest neighbours
3  1800.0       3.5    380.0   ← bedrooms imputed
4  2200.0       4.0    415.0   ← price imputed from neighbours
5  1900.0       5.0    520.0   ← sqft imputed

Iterative Imputation (MICE)

🔬 Scenario: Clinical trial data — multiple correlated lab measurements each with sporadic missingness. Use the full feature set to predict each missing value iteratively. 
from sklearn.experimental import enable_iterative_imputer  # noqa
from sklearn.impute import IterativeImputer
from sklearn.ensemble import RandomForestRegressor

lab = pd.DataFrame({
    "glucose":     [95, np.nan, 110, 88, np.nan, 102],
    "insulin":     [10, 12, np.nan, 9, 14, np.nan],
    "hba1c":       [5.2, 5.8, 6.1, np.nan, 6.5, 5.9],
})

mice = IterativeImputer(
    estimator=RandomForestRegressor(n_estimators=10, random_state=42),
    max_iter=10,
    random_state=42
)
lab_imputed = pd.DataFrame(
    mice.fit_transform(lab),
    columns=lab.columns
).round(2)
print(lab_imputed)
💡 Tip: MICE (Multiple Imputation by Chained Equations) is the gold standard for complex, correlated data. It models each feature's missingness as a function of all others, cycles multiple times, and produces statistically consistent imputations.

Missing Indicator — flag, then fill

🤖 Scenario: ML feature engineering — preserve the information that a value was missing, because missingness itself may be predictive (e.g. credit applications where income not stated). 
from sklearn.impute import MissingIndicator

credit = pd.DataFrame({
    "income":       [50000, np.nan, 75000, np.nan, 90000],
    "credit_score": [720, 650, np.nan, 580, 800],
    "loan_amount":  [10000, 15000, 8000, 20000, 5000],
})

# Create binary flags BEFORE filling
indicator = MissingIndicator(features="all")
flags = pd.DataFrame(
    indicator.fit_transform(credit),
    columns=[f"missing_{c}" for c in credit.columns],
    dtype="int8"
)

# Now fill and concatenate
credit_filled = credit.fillna(credit.median())
result = pd.concat([credit_filled, flags], axis=1)
print(result)

7. Real-World Pipeline

A complete, production-style function that audits, flags, and repairs a dataset — combining all techniques.

End-to-end missing-data handler for a retail sales dataset

import pandas as pd
import numpy as np

# ── Simulate a raw retail CSV ──────────────────────────────────
raw = pd.DataFrame({
    "order_id":   [1001, 1002, 1003, 1004, 1005, 1006, 1007],
    "customer":   ["Ana", "Ben", None, "Dan", "Eve", None, "Gil"],
    "category":   ["Books", "Books", "Electronics", None, "Clothing", "Electronics", "Books"],
    "amount":     [29.99, np.nan, 499.00, 15.00, np.nan, 299.00, 12.00],
    "discount":   [0.10, 0.05, np.nan, 0.00, 0.15, np.nan, 0.10],
    "order_date": ["2024-11-01", "2024-11-02", "2024-11-03", None,
                   "2024-11-05", "2024-11-06", None],
    "shipped":    ["2024-11-03", None, "2024-11-05", "2024-11-06",
                   None, "2024-11-08", "2024-11-09"],
})

def clean_sales(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()

    # ── Step 1: Audit ──────────────────────────────────────────
    print("=== MISSING AUDIT ===")
    audit = pd.DataFrame({
        "missing":    df.isnull().sum(),
        "pct":        (df.isnull().sum() / len(df) * 100).round(1),
        "dtype":      df.dtypes,
    })
    print(audit[audit["missing"] > 0])
    print()

    # ── Step 2: Parse dates ────────────────────────────────────
    df["order_date"] = pd.to_datetime(df["order_date"])
    df["shipped"]    = pd.to_datetime(df["shipped"])

    # ── Step 3: Flag missingness BEFORE imputation ─────────────
    df["amount_was_missing"]   = df["amount"].isna().astype("int8")
    df["discount_was_missing"] = df["discount"].isna().astype("int8")

    # ── Step 4: Drop records with no customer AND no order_date ─
    critical_missing = df["customer"].isna() & df["order_date"].isna()
    print(f"Dropping {critical_missing.sum()} rows with both customer and date missing.")
    df = df[~critical_missing].reset_index(drop=True)

    # ── Step 5: Fill known-pattern columns ────────────────────
    df["customer"]  = df["customer"].fillna("Guest")
    df["category"]  = df["category"].fillna("Uncategorized")
    df["discount"]  = df["discount"].fillna(0.0)    # no discount = 0%

    # ── Step 6: Group-median imputation for amount ─────────────
    df["amount"] = df.groupby("category")["amount"].transform(
        lambda x: x.fillna(x.median())
    )

    # ── Step 7: Forward-fill dates (ordered by order_id) ──────
    df = df.sort_values("order_id")
    df["order_date"] = df["order_date"].ffill(limit=1)
    df["shipped"]    = df["shipped"].bfill(limit=1)

    # ── Step 8: Derive shipping_days (NaT - safe with pd.Timedelta)
    df["shipping_days"] = (
        df["shipped"] - df["order_date"]
    ).dt.days

    print("=== AFTER CLEANING ===")
    print(df.isnull().sum())
    return df

clean = clean_sales(raw)
print(clean)

8. Decision Guide

Choose the right strategy based on your data and use-case.

1

Audit first

Run df.isnull().sum(), df.info(), and a percentage table. Understand why values are missing (random, systematic, or structural).

2

Drop if the column/row is beyond salvage

Use dropna() when >70% of a column is null, or when a row is missing every feature you need. Add subset= to target specific columns.

3

Fill with domain knowledge first

A missing discount means 0%, not the mean discount. Missing category is "Uncategorized". Use context before statistics.

4

Use median/mode for simple numeric/categorical imputation

Median is robust to outliers (better than mean for skewed distributions). For ML pipelines use SimpleImputer from scikit-learn.

5

Use ffill/bfill for time-series and ordered data

Always set limit= to avoid propagating stale values too far. Use interpolate(method="time") for uneven time gaps.

6

Use KNN or MICE for correlated multivariate data

Preserves inter-feature relationships. Use when features are correlated and you have enough complete rows to train from.

7

Add missingness indicator columns for ML

The fact that a value is absent is often predictive. Create feature_was_missing binary columns before imputing.
